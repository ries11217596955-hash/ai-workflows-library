---
title: Контракт запроса
---
Короткий "контракт запроса", чтобы результат ИИ был стабильным и проверяемым.
## Когда применять
Когда нужен повторяемый результат (документ, письмо, план, скрипт), особенно перед "выпуском / отправкой / публикацией".
## Шаблон контракта (копируй/вставь)
Заполняй только важное. Неизвестное помечай как **[ASSUMPTION]**.
- **Цель:** (что считается успехом)
- **Аудитория:** (кто будет читать/использовать)
- **Контекст:** (зачем / где будет применяться)
- **Вход:** (данные, заметки, ссылки, ограничения)
- **Формат выхода:** (заголовки, колонки таблицы, нейминг, объём, тон)
- **Обязательно включить:** (секции, поля, обязательные пункты)
- **Запрещено:** (спекуляции, чувствительные данные, неподтверждённые "факты")
- **Критерии приёмки:** (что делает результат "PASS")
- **Источники:** (даны / нет / нужны)
- **Допущения:** (явно)
- **Следующий шаг:** (что ты сделаешь с результатом)
## Рекомендуемый промпт
> Удобно использовать как первый запрос в любом workflow.
**ПРОМПТ**
1) Работай строго по контракту. Если чего-то нет - пометь **[ASSUMPTION]**.  
2) Вывод - строго в заданном формате.  
3) В конце: короткий чек - что подтверждено, что предположено.
