# ИИ для офисной работы

Хаб для офисных задач: письма, документы, таблицы, контроль действий.

## Офисный цикл (повторяемый)
1) Захватить вход (заметки / письмо / файл).
2) Нормализовать (почистить текст + выделить факты).
3) Сформировать результат (письмо / мемо / таблица).
4) Проверить (чек-лист качества).
5) Сохранить или отправить.

## Самые полезные сценарии

### 1) Разбор почты → ответы
Use case: [Email-ассистент](/ru/use-cases/email/)

**Промпт**
```text
Ты мой email-ассистент.
Цель: подготовить ответ.
Ограничения: <тон, объём, что обязательно включить>.
Входное письмо:
<ВСТАВЬ>
Выход: 2 варианта ответа + тема письма.
```

### 2) Заметки → план действий (таблица)
Use case: [Встреча → план действий](/ru/use-cases/meeting-to-action-plan/)

**Промпт**
```text
Преобразуй заметки в таблицу плана действий.
Колонки: Действие | Ответственный | Срок | Статус | Примечания
Заметки:
<ВСТАВЬ>
```

### 3) Приведение документа (структура + формат)
**Промпт**
```text
Приведи документ в порядок.
Правила:
- смысл сохранить
- убрать повторы
- добавить заголовки где нужно
Выход: Markdown с заголовками + короткое резюме.
Текст:
<ВСТАВЬ>
```

### 4) Очистка таблицы (гигиена данных)
**Промпт**
```text
Очисти табличные данные.
Сделай:
- нормализация дат
- унификация наименований
- поиск дублей
Выход:
1) очищенная таблица (те же колонки)
2) список найденных проблем
Таблица:
<ВСТАВЬ>
```

## Быстрые шаблоны

### Служебная записка
```text
Напиши служебную записку.
Тема: <тема>.
Аудитория: <кто>.
Разделы: Контекст / Решение / Действия / Риски.
Объём: до <N> слов.
```

### Итог встречи
```text
Суммируй встречу.
Выход:
1) Резюме (5 буллетов)
2) Решения
3) Действия (таблица)
Заметки:
<ВСТАВЬ>
```

## Гварды
- Не придумывай даты, имена, цифры.
- Если неизвестно — пометь как [CHECK].
- Перед отправкой прогони [Чек-лист качества](/ru/tools/quality-checklist/).
