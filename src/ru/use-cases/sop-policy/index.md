# SOP / политика / регламент

**Цель:** Написать практичный регламент, которому можно следовать.

## Что нужно на вход (inputs)
- Название процесса
- Границы (что входит/не входит)
- Роли
- Входы/выходы
- Риски (опционально)

## Готовые промпты (копипаст)
**Промпт 1 — Черновик SOP**

```text
Сделай SOP/регламент для процесса: <процесс>.
Включи:
- Цель
- Границы (входит/не входит)
- Определения (если нужно)
- Роли и ответственность
- Пошаговая процедура
- Записи/логи
- Проверки качества
Ограничения: просто, без юридических обещаний.
Вход:
<<<INPUTS>>>
```

**Промпт 2 — Версия чек‑листа**

```text
Преобразуй SOP в формат чек‑листа для ежедневного использования.
SOP:
<<<SOP>>>
```

**Промпт 3 — Failure modes**

```text
Перечисли 10 типовых сбоев этого SOP и добавь проверки предотвращения.
SOP:
<<<SOP>>>
```


## Проверка качества (30 секунд)
- Границы процесса ясны
- Шаги исполнимы
- Есть проверки
- Нет нереалистичных обещаний

## Типовые ошибки
- Слишком абстрактно
- Нет ролей
- Нет раздела записей/логов
