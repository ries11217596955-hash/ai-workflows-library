# Встреча → план действий

Цель: превратить заметки/стенограмму встречи в **чёткий план действий**: решения, задачи, ответственные, сроки и риски.

## Входные данные (скопируй и заполни)
- Тема встречи:
- Дата/время:
- Участники (имя/роль):
- Заметки или стенограмма (вставь ниже):
- Ограничения: (дедлайн, бюджет, инструменты, “нельзя/обязательно” и т.п.)
- Стиль результата: (строго / дружелюбно / для руководителя)
- Часовой пояс сроков: (например, UTC+4)

## Промпт (вставь в ИИ)
Роль: ты — операционный менеджер.  
Задача: преобразуй заметки во **внятный план действий**. Факты **не выдумывать**. Если чего-то не хватает — фиксируй как вопрос в разделе “Открытые вопросы”.

Формат результата (строго в этом порядке):

1) **Краткое резюме (до 5 буллетов)**
2) **Принятые решения**
   - Решение → обоснование (если было) → эффект
3) **Таблица плана действий**
   - Задача | Владелец | Срок | Приоритет (H/M/L) | Зависимости | Ссылка/доказательство (если есть)
4) **Риски и блокеры**
   - Риск/блокер → почему важно → мера → владелец
5) **Открытые вопросы (обязательны к уточнению)**
6) **Повестка следующей встречи (10–20 минут)**
7) **Черновик сообщения участникам** (короткое письмо/Slack с задачами и сроками)

Правила:
- Если сроки не указаны — ставь “TBD” и продублируй в “Открытые вопросы”.
- Если владелец не указан — ставь “Не назначен” и предложи роль (не человека).

## Быстрая проверка качества
- Каждая задача — действие (глагол + объект)?
- У каждой задачи есть владелец (или “Не назначен”)?
- Сроки есть (или честно TBD)?
- У рисков есть мера и владелец?
- Нет выдуманных фактов?
