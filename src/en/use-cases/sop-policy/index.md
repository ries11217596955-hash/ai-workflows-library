# Draft a SOP / policy

**Goal:** Write a practical SOP that people can follow.

## What you need (inputs)
- Process name
- Scope (what’s included/excluded)
- Roles involved
- Inputs/outputs
- Risks (optional)

## Copy‑paste prompt templates
**Prompt 1 — SOP draft**

```text
Draft a SOP for this process: <process>.
Include:
- Purpose
- Scope (in/out)
- Definitions (if needed)
- Roles & responsibilities
- Step-by-step procedure
- Records/logs
- Quality checks
Constraints: simple, no legal claims.
Inputs:
<<<INPUTS>>>
```

**Prompt 2 — Checklist version**

```text
Convert this SOP into a checklist format for daily use.
SOP:
<<<SOP>>>
```

**Prompt 3 — Failure modes**

```text
List top 10 failure modes for this SOP and add prevention checks.
SOP:
<<<SOP>>>
```


## Quality check (30 seconds)
- Scope is explicit
- Steps are executable
- Checks exist
- No unrealistic promises

## Common mistakes
- Too abstract
- No roles
- No records/logs section
